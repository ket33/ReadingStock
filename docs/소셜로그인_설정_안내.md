# 로그인 설정 안내 — 비개발자용 (구글 + 이메일/비밀번호)

코드는 준비돼 있어요. 아래만 마치면 로그인이 살아납니다.
**총 4단계**: ① 관심종목 테이블 → ② 구글 키 → ③ 메일 발송(SMTP) 연결 → ④ 주소 등록
(카카오 로그인은 **보류** — 나중에 켜고 싶으면 말씀해 주세요. 버튼만 다시 달면 됩니다)

> 아래에서 계속 쓰는 값 (미리 복사해 두세요)
> **콜백 주소**: `https://ajrrsaslyveqwdusoxbz.supabase.co/auth/v1/callback`

## ① 관심종목 테이블 만들기 (5분)

1. https://supabase.com/dashboard → 프로젝트 선택
2. 왼쪽 **SQL Editor** → `web/sql/watchlist.sql` 내용 전체 붙여넣기 → **Run**
3. 결과에 정책 3개(`watchlist_select_own / insert_own / delete_own`)가 보이면 성공

---

## ② 구글 로그인 키 (15분)

1. https://console.cloud.google.com → 구글 계정 로그인
2. 상단 프로젝트 선택 → **새 프로젝트** (이름 `Reading Stock`) → 만들기 → 그 프로젝트 선택
3. **API 및 서비스 > OAuth 동의 화면**
   - User Type **외부(External)** → 만들기
   - 앱 이름 `Reading Stock`, 사용자 지원 이메일·개발자 연락처 = 본인 이메일 → 저장
   - (범위·테스트 사용자는 기본값으로 계속)
4. **API 및 서비스 > 사용자 인증 정보** → **+ 만들기 > OAuth 클라이언트 ID**
   - 유형 **웹 애플리케이션**, 이름 `readingstock`
   - **승인된 리디렉션 URI** = 위의 콜백 주소 → 만들기
5. 나오는 **클라이언트 ID / 보안 비밀번호** 복사
6. Supabase → **Authentication > Sign In / Providers > Google**
   - Enabled **ON** → Client ID / Secret 붙여넣기 → Save
7. ⚠ 동의 화면이 "테스트" 상태면 본인만 로그인돼요.
   **OAuth 동의 화면 > 게시 상태 > 앱 게시**를 눌러야 모두 로그인 가능.

---

## ③ 메일 발송(SMTP) 연결 — 가입 확인·비밀번호 재설정 메일용 (20분)

Supabase 기본 메일은 **시간당 2통(테스트용)**이라, 실서비스용으로 무료 발송 서비스(Resend)를 연결합니다.
(무료 한도: 월 3,000통 / 일 100통 — 지금 규모엔 충분)

1. https://resend.com 가입 (구글 계정으로 가능)
2. 대시보드 → **Domains** → **Add Domain** → `readingstock.com` 입력
3. 화면에 **DNS 레코드 몇 개**(TXT·MX 형태)가 나와요 → 이걸 도메인 관리처에 추가:
   - 도메인을 **Vercel**에서 샀다면: Vercel 대시보드 → **Domains** → readingstock.com → **DNS Records** → 화면에 나온 레코드를 하나씩 그대로 추가
   - 다른 곳(가비아 등)이면 그곳의 DNS 관리 메뉴에서 추가
4. Resend 화면에서 **Verify** 클릭 → 몇 분 내 초록불(Verified) 확인
5. Resend → **API Keys** → **Create API Key** (권한 Sending) → 키 복사
6. Supabase → **Project Settings > Authentication**(또는 Authentication > Emails > **SMTP Settings**) → **Enable Custom SMTP**:
   - Host: `smtp.resend.com`
   - Port: `465`
   - Username: `resend`
   - Password: (5번에서 복사한 API 키)
   - Sender email: `no-reply@readingstock.com`
   - Sender name: `Reading Stock`
   - Save
7. (권장) Supabase → **Authentication > Rate Limits**에서 이메일 발송 한도를 시간당 30 이상으로 상향

> 급하면 ③을 건너뛰어도 로그인 자체는 동작해요. 다만 가입 확인 메일이 시간당 2통까지만 나가요.

---

## ④ 사이트 주소 등록 (2분)

Supabase → **Authentication > URL Configuration**
- **Site URL**: `https://readingstock.com`
- **Redirect URLs**:
  - `https://readingstock.com/**`
  - `http://localhost:3100/**` (로컬 테스트용)

또 하나 확인: **Authentication > Sign In / Providers > Email**이 **Enabled**인지 (기본값 ON).

---

## 완료 확인 (배포 후)

1. 사이트 우상단 **시작하기** → 회원가입 탭 → 동의 체크 → 이메일+비밀번호 가입
2. 가입 확인 메일의 링크 클릭 → 로그인됨 (우상단에 이메일 표시)
3. 종목 페이지 **☆ 관심종목** 클릭 → ★ 담김 → 우상단 메뉴 **내 관심종목**에서 확인·삭제
4. 로그인 탭 **비밀번호를 잊으셨나요?** → 재설정 메일 → 새 비밀번호 설정 동작 확인
5. Google 버튼으로도 가입·로그인 확인
6. (보안 테스트) 두 번째 계정으로 로그인 → 첫 계정의 관심종목이 **안 보여야** 정상
